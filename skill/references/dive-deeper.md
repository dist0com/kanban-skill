# Dive deeper

Take one task and move it one step forward — from vague to concrete. Read the card,
judge where it stands, then push it to the next stage. Update the card in place with
what you find.

Read the card first. Judge how far along it is, then take the next step:

- **A rough idea** → work out if it's worth doing and how. The user value, who it's for,
  whether we can build it, and the open questions (e.g. different ways the user could
  experience it). Land on a clearer shape.
- **A decided-on feature** → design the exact user experience. What the user sees, each
  step they take, what happens on edge cases.
- **A research or content task** → find the specific question to answer or the piece's
  structure, and what we can say that others don't.

These are examples, not a fixed list. Read the task, decide what "one step forward"
means for it, and take that step. Don't jump to the end — go one stage, not three.

**Stop at the code level.** Never dive deeper once the plan is concrete. Don't
mention coding details — what functions to add, what files to touch. Stop when the
whole plan is easy to start.

Write the result back into the card. If a step raises a question only the user can
answer, ask it before guessing.
